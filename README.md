# CS-330-Portfolio

Portfolio repository for CS 330 Computational Graphics and Visualization.

## Portfolio Artifacts

This repository includes my final CS 330 project files and my final project submission document. My project is a 3D desk-style scene with a wooden tabletop, notebook, saucer, coffee mug, and coffee surface. These files show my work with 3D objects, transformations, textures, materials, lighting, and camera navigation.

## Reflection

### How do I approach designing software?

When I design software, I like to start by thinking about what the final project is supposed to look like and what the user should be able to experience. For my CS 330 project, I started with a simple tabletop scene idea and broke it down into smaller objects like the desk, mug, saucer, coffee, and notebook. This helped me stay organized instead of trying to build the entire scene all at once.

This project helped me build better design skills because I learned how to plan a 3D scene using basic shapes, textures, lighting, and transformations. I had to think about object placement, size, rotation, and how each object fit with the rest of the scene. I also learned that design is not only about making something look nice. It is also about making sure the project is organized and easy to understand in the code.

The design process I followed was step-by-step. I started by choosing the objects for my scene, then I created the basic shapes, adjusted their positions, added textures, and later improved the lighting and materials. In future work, I can use this same approach by planning first, building in smaller sections, testing often, and improving the project as I go.

### How do I approach developing programs?

When I develop programs, I usually work in small parts and test often. For this 3D scene, I built the project through the milestones instead of trying to finish everything in one sitting. I first focused on getting the objects into the scene, then added camera movement, textures, lighting, and materials.

One development strategy I used was iteration. I would make a change, run the program, look at what happened, and then adjust it if something looked wrong. For example, I had to keep adjusting the mug and saucer because small changes in the X, Y, and Z positions made a big difference. Sometimes the mug looked like it was floating or clipping into the saucer, so I had to fix the placement until it looked more natural.

My approach to coding has improved throughout the milestones because I became more comfortable testing and improving my work instead of expecting everything to be perfect the first time. At the beginning, I was mostly focused on just making objects appear. By the end, I was thinking more about how the whole scene worked together, including textures, lighting, materials, and navigation controls.

### How can computer science help me in reaching my goals?

Computer science can help me reach my goals because it teaches me how digital projects are created behind the scenes. I am interested in 3D animation, so learning computational graphics helped me understand more about how 3D objects, lighting, textures, and scenes are built through code.

Computational graphics and visualization gave me skills that can help in my future classes because I now understand how important transformations, camera movement, textures, and lighting are in a 3D environment. These skills can also help me in future game development or animation-related projects because I have a better idea of how visual scenes are created and controlled.

Professionally, this knowledge can help me because creative fields like animation, games, simulations, and visual design all use graphics concepts. Even if I work more on the artistic side in the future, understanding the technical side will help me communicate better with programmers, designers, and other team members. This project helped me see how computer science connects to creative work, which is important for my future career goals.
